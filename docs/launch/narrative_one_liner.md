# Narrative One-Liners

Use one primary line and one supporting line. Do not switch category language mid-launch.

## Core Category Line

Gait is the offline-first policy-as-code runtime for AI agent tool calls.

## Product Promise Line

Prove the install fast, enforce at the tool boundary when you own the seam, and graduate to hardened `oss-prod` readiness explicitly.

## Audience Variants

Platform engineering:

- Control agent tool execution at the boundary, not after the fact.

Security:

- Enforce least-privilege policy on tool calls with signed, auditable traces.

Developer:

- `gait init` and `gait check` to a real tool-boundary regression path in minutes, with no hosted dependency.

Compliance:

- Evidence packs and stable schemas designed for tickets, audits, and change control.

## Hard Boundaries (must stay explicit)

- Not an agent builder or orchestrator.
- Not prompt-only filtering.
- Not a hosted governance dashboard requirement.
- Quickstart or demo proof is not hardened `oss-prod` readiness.
- Strict inline fail-closed enforcement requires interception before tool execution.
